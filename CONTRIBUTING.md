# Contributing to SignalWire Academy

Thank you for your interest in contributing to SignalWire Academy! This document provides guidelines for contributing to our courseware and template repositories.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. Check existing issues to avoid duplicates
2. Create a new issue with a clear title and description
3. Include steps to reproduce (for bugs)
4. Add relevant labels if available

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-improvement`)
3. Make your changes
4. Test your changes thoroughly
5. Commit with clear, descriptive messages
6. Push to your fork
7. Open a Pull Request

### Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Include docstrings for functions and classes
- Keep functions focused and concise

### Testing

- Ensure all existing tests pass before submitting
- Add tests for new functionality
- Use `swaig-test` to validate agent configurations

## Course Content Contributions

For courseware content (documentation, tutorials):

- Use clear, beginner-friendly language
- Include code examples where appropriate
- Test all code snippets before submitting
- Follow existing formatting conventions

## Questions?

See our [Support Guide](SUPPORT.md) for how to get help.
