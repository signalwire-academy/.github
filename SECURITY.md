# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in SignalWire Academy courseware or templates, please report it responsibly.

### How to Report

**Do NOT create a public GitHub issue for security vulnerabilities.**

Instead, please email: security@signalwire.com

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 5 business days
- **Resolution Timeline**: Varies based on severity

### Scope

This security policy applies to:
- All `signalwire-academy` repositories
- Course content and example code
- Template repositories

### Out of Scope

- SignalWire platform/API security (report to SignalWire directly)
- Third-party dependencies (report to maintainers)

## Best Practices for Students

When working on labs and assignments:

- Never commit API keys or credentials
- Use environment variables for sensitive data
- Review `.gitignore` before committing
- Do not expose endpoints publicly without authentication

Thank you for helping keep SignalWire Academy secure.
